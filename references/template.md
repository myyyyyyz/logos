# 日志摘要模板

## 新建文件头

仅当 `logos/your-memory/YYYY-MM-DD.md` 不存在时写入：

```markdown
# {YYYY}年{MM}月{DD}日

> 自动生成于 {生成时间} | 由 Logos 驱动
```

## 对话摘要段落

每次对话记录使用以下格式。追加时先写 `---`，再写段落。**重要：各区块按重要性从上到下排列。**

```markdown
---

### {开始时间} ~ {结束时间} | 第 {N} 次对话

#### 💎 灵感与收获
{这是最重要的段落。逐条列出本轮对话中值得记住的东西，标注来源}

**来自用户：**
- {用户提出的巧妙想法、设计思路、对未来方向的构想}

**来自 AI：**
- {AI 推荐的最佳实践、反模式警告、性能优化技巧、被验证有效的方案}

**踩坑提醒：**
- {如果发现了某个坑、某个配置陷阱、某个容易犯的错误}

#### 🎯 关键决策
| 决策 | 原因 |
|------|------|
| {决策1} | {为什么这样做} |
| {决策2} | {为什么这样做} |

#### 📋 需求概述
{一句话描述本轮对话的核心目标}

#### 📝 对话摘要
{3-5 句话概括对话过程，侧重做了什么、解决了什么}

#### 📦 产物清单
- `新建` `{文件路径}` - {文件作用}
- `修改` `{文件路径}` - {改了什么}

如果没有文件变更，写：
- 无文件变更

#### ✅ 验证结果
- `{命令}` - {通过 / 失败 / 未运行；关键输出或失败原因}

#### ⚠️ 待办 / 遗留
- [ ] {待办项1}
- [ ] {待办项2}

如果没有待办，写：
- 无
```

## 追加规则

- 当日已有内容时只追加段落，不重复写文件头。
- 段间分隔线必须是单独一行 `---`。
- `第 {N} 次对话` 从已有最大编号继续递增。
- 不确定的信息不猜测，写入"待办"。
- 敏感信息用 `[REDACTED]` 替代。
- **"💎 灵感与收获"不允许为空**——如果本轮确实没有新发现，写"本轮未产生新的灵感或发现"。但如果对话中有值得记住的东西，必须写清楚。
