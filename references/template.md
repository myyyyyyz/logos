# 日报摘要模板

## 新建文件头

仅当 `docs/daily/YYYY-MM-DD.md` 不存在时写入：

```markdown
# {YYYY}年{MM}月{DD}日 工作日报

> 自动生成于 {生成时间} | 由 Logos 驱动
```

## 对话摘要段落

每次对话记录使用以下段落格式。追加到已有日报时，先写入单独一行 `---`，再写入段落标题。

```markdown
---

### {开始时间} ~ {结束时间} | 第 {N} 次对话

#### 需求概述
{一句话描述本轮对话的核心目标}

#### 关键决策
| 决策 | 原因 |
|------|------|
| {决策1} | {为什么这样做} |
| {决策2} | {为什么这样做} |

#### 对话摘要
{3-5 句话概括整个对话过程，重点写做了什么、解决了什么问题}

#### 产物清单
- `新建` `{文件路径}` - {文件作用}
- `修改` `{文件路径}` - {改了什么}
- `删除` `{文件路径}` - {为什么删除}

如果没有文件变更，写：

- 无文件变更

#### 验证结果
- `{命令}` - {通过 / 失败 / 未运行；关键输出或失败原因}

#### 待办 / 遗留
- [ ] {待办项1}
- [ ] {待办项2}

如果没有待办，写：

- 无

#### 经验沉淀
{如果有值得记住的经验、新学到的技术、踩过的坑，写在这里；没有则写“无”}
```

## 追加规则

- 当日已有内容时，只追加“对话摘要段落”，不要重复写文件头。
- 段间分隔线必须是单独一行 `---`。
- `第 {N} 次对话` 从已有最大编号继续递增。
- 不确定的信息不要猜测，写入“待办 / 遗留”。
- 敏感信息用 `[REDACTED]` 替代。
