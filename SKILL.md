# Logos — AI 对话沉淀与日报生成

## 核心原则

每一次 AI 对话都是知识资产，不能"聊完就忘"。本轮对话的所有内容（用户提问 + AI 回答 + 产生的代码/决策）都是输入，AI 负责提炼、归类、归档。

硬性约束：
- 写文件前先确认 `docs/daily/` 目录存在，不存在则创建
- 不得编造对话中没出现的内容
- 不得遗漏用户明确要求记住的信息

## 工作流（三步，不可跳过）

### 第一步：回顾对话

从当前上下文窗口中的完整对话历史，提取以下信息：

1. **用户原始需求**：每条用户消息的核心诉求
2. **AI 做了什么**：代码修改、文件创建、搜索、设计决策
3. **关键决策**：技术选型、架构调整、方案取舍及原因
4. **产物清单**：新建/修改了哪些文件，做了什么
5. **待办/遗留问题**：用户提到但未处理的事项

### 第二步：AI 生成摘要

将第一步提取的信息，按模板生成结构化摘要。模板见 `references/template.md`。

关键要求：
- 每段摘要控制在 20-50 行
- 技术决策必须附上"为什么"
- 产物清单用文件路径列表
- 如果当日文件已存在，追加到已有内容末尾（用 `---` 分隔）

### 第三步：写入文件

写入 `docs/daily/YYYY-MM-DD.md`。

规则：
- 如果当日文件不存在 → 创建新文件，写入模板头 + 首段摘要
- 如果当日文件已存在 → 追加到文件末尾，段间用 `---` 分隔线
- 文件路径：`{项目根目录}/docs/daily/YYYY-MM-DD.md`

## 自动触发机制

为了让 Skill 在每次对话结束后自动执行，需要借助 AI 工具的 Hook 或定时任务能力。

### 方案 1：Hook 触发（推荐）

如果 AI 工具支持 SessionEnd / Stop hook，在对话结束时自动调用本 Skill。参考配置：

```json
{
  "hooks": {
    "Stop": [
      {
        "matcher": "",
        "command": "提示 AI 执行 logos skill 总结本轮对话"
      }
    ]
  }
}
```

### 方案 2：定时调度

通过 AI 工具的定时任务功能，设置每日定时触发（如 19:00），对本日所有对话做总结归档。

### 方案 3：手动触发

对话末尾直接说：
- "记录本次对话"
- "沉淀今天的内容"
- "/logos"

Skill 会立即执行三步工作流。

## 并行策略

如果当日已有多次对话记录，首轮并行读取：
- `docs/daily/YYYY-MM-DD.md`（已有内容）
- 上下文窗口中的完整对话历史

然后合并生成追加内容。

## 不做什么

- 不编造对话中没发生的操作或决策
- 不跳过用户的任何一条需求
- 不省略"为什么"——每个技术决策都要记录理由
- 不把多天的内容混到同一个文件
- 不在纯闲聊中强制记录（如果对话无实质内容，告知用户"本次对话无实质技术内容，建议跳过"）
- 不覆盖已有日报——总是追加
